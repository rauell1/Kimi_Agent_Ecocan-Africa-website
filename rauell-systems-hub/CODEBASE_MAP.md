## Key Config Files

### `app.config.ts`
TanStack Start server config. Sets `preset: 'vercel'` so Vinxi outputs a Node.js-compatible `dist/server/server.js`.

### `vite.config.ts`
Uses `tanstackStart()` from `@tanstack/react-start/plugin/vite` (native plugin — not the Lovable wrapper). Also loads `vite-tsconfig-paths` for `@/` alias resolution.

### `vercel.json`
```json
{
  "buildCommand": "npm run build && npm run build:api",
  "outputDirectory": "dist/client",
  "routes": [
    { "src": "/assets/(.*)", "dest": "/assets/$1" },
    { "src": "/(.*)", "dest": "/api/ssr" }
  ]
}
```
Static assets pass through; all other routes hit the SSR serverless function.

### `scripts/build-api.mjs`
Runs after `vite build`. Uses esbuild to bundle `dist/server/server.js` (the TanStack Start SSR output) into `api/ssr.js` — a self-contained Vercel serverless function with all dependencies inlined.

## Build Pipeline

```
npm run build
    └── vite build
            ├── client build  →  dist/client/   (static assets + JS chunks)
            └── SSR build     →  dist/server/server.js  (Node.js SSR handler)

npm run build:api
    └── scripts/build-api.mjs
            └── esbuild bundles dist/server/server.js → api/ssr.js
                (platform: node, target: node20, format: esm, packages: bundle)
```
